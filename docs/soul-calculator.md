# 凝魂升格 & 魂力计算

页面：`flipgame/SoulAscensionCalculator.html`  
访问权限：注册会员。页面加载时会调用 `/api/me` 检查当前登录账号，禁用账号不可访问。  
主数据：`flipgame/soul_tiers.csv`  
价格数据：`flipgame/quality_prices.json`，直接维护 JSON。
兼容候选：`凝魂.csv`

## CSV 字段口径

- `级别`：资质段名称，例如 `sss`、`X`、`X Full`。
- `平均分值`：该资质段三维均值下限。
- `魂点上限`：该资质段魂力上限。
- `升格需求`：当前资质段升格材料配方，用于升格成本。
- `基础凝魂` + `奖励凝魂`：吃该资质提供的凝魂经验。
- `凝魂进度`：每次凝魂需要的经验。
- `凝魂强化`：每次凝魂增加的三维总额。
- `强化次数`：该段可强化次数，用于推导最后一段上限。

## 三维计算

```text
当前总和 = x + y + 100*z
当前均值 = (x + y + 100*z) / 3
```

凝魂次数可以按总和或均值计算，两者等价：

```text
次数 = ceil((目标总和 - 当前总和) / 每次总和增量)
次数 = ceil((目标均值 - 当前均值) / 每次均值增量)
每次均值增量 = 凝魂强化 / 3
```

## 速度隐藏小数

游戏界面速度可能只显示整数。页面按 `z` 和 `z+1` 两组结果计算：

- 次数不同：显示范围。
- 魂力不同：显示范围。
- 上下限相同：显示单值。

## SSS 与 X 规则

旧上限：

```text
x <= 9900
y <= 9900
z <= 99
```

当前 X 规则：

- `X` 起点在 CSV 中为 `9901`。
- 任意一维超过旧上限时，进入 X 输入校验。
- 有效 X 输入必须满足 `x > 9900`、`y > 9900`、`z >= 99`。
- 若任意一维超过旧上限，但三维没有同时满足 X 条件，页面显示 `X 数值错误`。
- `9900, 9900, 99` 固定视为 SSS 顶，不进入 X。
- 由于速度隐藏小数，`z=99` 可能通过 `z+1` 分支覆盖 X 计算。

X 目标均值：

```text
X目标均值 = 9900 + 强化次数 * 凝魂强化 / 3
```

当前表中：

```text
X目标均值 = 9900 + 100 * 60 / 3 = 11900
```

`X Full` 行用于记录完整目标均值，不作为可凝魂段参与计算。

## 魂力计算

魂力在当前资质段内线性插值：

```text
当前魂力 = 当前段魂力下限 + 区间比例 * (当前段魂点上限 - 当前段魂力下限)
区间比例 = (当前均值 - 当前段均值下限) / (当前段均值上限 - 当前段均值下限)
```

当前段魂力下限等于上一资质段 `魂点上限`。

X 当前实测推导：

- `X` 魂点上限使用 `13900`。
- 该值来自实测点 `11513, 11605, 116` 对应魂力 `13259` 后反推，并取整为当前维护值。

## 升格成本

页面从当前资质段 `升格需求` 读取材料配方，并使用 `quality_prices.json` 中对应资质的 `foodPrice` 计算升格成本。

示例：

```text
sss 升格需求 = 1s,1ss,1sss
```

如果材料价格缺失，页面提示价格数据缺少对应升格材料单价，不显示总价。

## 凝魂性价比

页面使用 `quality_prices.json` 的 `foodPrice`，以及 CSV 中该资质的凝魂经验来源，计算达到所需总经验的最低成本喂法。

价格 JSON 中不存在或无效的资质会被忽略。

为了避免出现单一低价资质堆叠过多的喂法，最优组合搜索限制每种资质最多使用 `10` 个。
