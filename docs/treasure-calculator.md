# 珍宝升级计算器

页面：`flipgame/TreasureCalculator.html`

原始图：`flipgame/images/珍宝.jpg`

## 数据口径

原始图标题为“珍宝升级碎片统计”，按珍宝颜色、升星阶段和小等级记录碎片消耗。1 星本身也需要碎片，`0星 -> 1星` 没有小阶段，直接消耗与 `1星 -> 2星` 合计相同的碎片。原图里的“升二星”表示当前是 1 星、正在升到 2 星；“升五星”表示当前是 4 星、正在升到 5 星；`4星 -> 5星` 到 `10` 级就是五星满级。

当前整理出三组颜色：

- `粉橙`：粉色 / 橙色珍宝共用同一组消耗。
- `红`：红色珍宝。
- `绿`：绿色珍宝。

阶段最高小等级：

- `0星 -> 1星`：无小阶段，直接消耗一笔碎片
- `1星 -> 2星`（原图：升二星）：`0 -> 3`
- `2星 -> 3星`（原图：升三星）：`0 -> 5`
- `3星 -> 4星`（原图：升四星）：`0 -> 7`
- `4星 -> 5星`（原图：升五星）：`0 -> 10`

## 计算规则

页面按“当前阶段 + 当前阶段小等级”到“目标阶段 + 目标阶段小等级”求区间和。这里的阶段使用实际星级变化，例如 `0星 -> 1星`、`1星 -> 2星`，不直接用原图的“升二星”作为主输入文案。

同一阶段内：

```text
需要碎片 = sum(cost[level] for level in 当前小等级 .. 目标小等级 - 1)
```

跨阶段时：

```text
需要碎片 = 当前阶段剩余 + 中间阶段满级 + 目标阶段部分小等级
```

如果目标点早于当前点，页面显示错误提示，不计算负数。
