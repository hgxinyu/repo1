# 觉醒冲榜模拟器

页面：`flipgame/AwakeningRushSimulator.html`

访问权限：VIP 专属。页面加载时会调用 `/api/me` 检查当前登录账号是否具备 `vip` 或管理员权限。

## 规则前提

- 每次觉醒消耗 `100` 绑钻。
- 每次觉醒先计入冲榜分数。
- 觉醒后，`B+` 及以下资质全部分解并返还绑钻。
- `A-`、`A`、`A+`、`S`、`SS`、`SSS` 资质保留，不分解、不返钻。
- 返还的绑钻继续用于后续觉醒。
- 绑钻不足 `100` 时停止模拟。
- 页面输入绑钻数量，最大值为 `2,000,000`。

## 期望值

单次觉醒期望分数：

```text
sum(概率 * 对应分数) = 5.92963
```

单次觉醒期望返钻：

```text
sum(概率 * 分解得钻) = 47.731
```

返钻率：

```text
47.731 / 100 = 0.47731
```

有效觉醒次数：

```text
初始绑钻 / (100 - 47.731)
```

理论期望总分：

```text
有效觉醒次数 * 5.92963
```

示例：`50,000` 绑钻。

```text
有效觉醒次数 = 50000 / (100 - 47.731) = 956.589948
理论期望总分 = 956.589948 * 5.92963 = 5672.224454
```

## 随机模拟输出

页面的随机模拟按逐抽流程执行：

```text
while 当前绑钻 >= 100:
  当前绑钻 -= 100
  随机抽取资质
  总分 += 资质分数
  if 资质 <= B+:
    当前绑钻 += 分解得钻
  else:
    A-以上保留数量 += 1
```

主要输出：

- 总分。
- 剩余的 `A-` 以上资质数量。

辅助输出：

- 觉醒次数。
- 停止时剩余绑钻。
- `A-`、`A`、`A+`、`S`、`SS`、`SSS` 分档数量。
- 理论期望分数和理论期望 `A-` 以上数量，用于对比随机波动。
